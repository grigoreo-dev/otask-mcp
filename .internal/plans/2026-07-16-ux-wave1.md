# UX Wave 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use beads-superpowers:subagent-driven-development (recommended) or beads-superpowers:executing-plans to implement this plan task-by-task. Each Task becomes a bead (`bd create -t task --parent <epic-id>`). Steps within tasks use checkbox (`- [ ]`) syntax for human readability.

**Goal:** Ship Worker landing + O!task icon + 2-step OAuth wizard (auth → scope picks via OAUTH_KV pending) + `otask_list_workspaces` + single-пространство auto-default + disclaimer docs.

**Architecture:** Extend Worker `AuthHandler` (OAuthProvider `defaultHandler`) for `/`, favicon, and wizard. Pending login lives in `OAUTH_KV` with a tiny HMAC-signed cookie (`pendingId` only). Core adds `listTeams` API + tool; `resolveWsSlug` becomes async with single-team auto-default.

**Tech Stack:** Bun, TypeScript, Cloudflare Workers, `@cloudflare/workers-oauth-provider`, existing `@grigoreo-dev/otask-mcp-core`, bun:test, Biome.

**Spec:** `.internal/specs/2026-07-16-ux-wave1-design.md` (stress-tested)

## Global Constraints

- UI RU term: **пространство** (not “workspace” in user-facing copy); code keeps `ws` / `defaultWs`.
- Never put O!task Bearer in cookie, HTML, or query; pending token only in KV ≤5m.
- Do not rewire OAuthProvider library routes (`/oauth/*`, well-known, `/mcp`); only `defaultHandler`.
- Empty multi-select allow-lists = unrestricted (same as today) + helper text.
- No `OTASK_*` secrets on public Worker env.
- Disclaimer on landing, wizard, README.
- TDD: failing test → implement → pass → commit per task.
- Run `bun test` and `bun run lint` before claiming task done.

- [ ] **Step 3: Implement `resolveWsSlug`**

```ts
export async function resolveWsSlug(
  ws_slug: string | undefined,
  scope: ScopeContext,
  listWorkspaces?: () => Promise<Array<{ slug: string }>>
): Promise<string> {
  let ws = trimOrUndef(ws_slug) ?? scope.defaultWs;
  if (!ws) {
    if (listWorkspaces) {
      const teams = await listWorkspaces();
      if (teams.length === 1 && teams[0]?.slug) {
        ws = teams[0].slug;
      }
    }
  }
  if (!ws) {
    throw new Error(
      "ws_slug is required (pass arg or set OTASK_DEFAULT_WS / X-Otask-Default-Ws). Call otask_list_workspaces, then pass ws_slug, or reconnect OAuth with a default workspace."
    );
  }
  scope.wsGuard.assertAllowed(ws);
  return ws;
}
```

export function createListWorkspacesTool({

Use when ws_slug is missing or to discover slugs before other tools.

Returns compact: id, slug, name.

Docs: https://api.otask.ru/docs`,

- [ ] **Step 3: Implement `pending-auth.ts`**

Design:

```ts
export const PENDING_COOKIE = "otask_mcp_pending";
export const PENDING_TTL_SEC = 300;
const KEY_PREFIX = "pending:v1:";
const HMAC_CONTEXT = "pending-v1:";

export type PendingAuth = {
  otaskToken: string;
  email: string;
  oauthReqJson: string; // JSON.stringify(oauthReq) from parseAuthRequest
  fingerprint: {
    clientId?: string;
    redirectUri?: string;
    codeChallenge?: string;
  };
  exp: number; // unix seconds
};

export function createPendingId(): string {
  return crypto.randomUUID();
}

export function pendingKvKey(id: string): string {
  return `${KEY_PREFIX}${id}`;
}

// sign: `${id}.${hexHmac}` using same WebCrypto HMAC as user-id.ts, message = HMAC_CONTEXT + id
// verify: parse cookie header, split, verify HMAC in constant-time-ish compare
// putPending: kv.put(key, JSON.stringify(value), { expirationTtl: PENDING_TTL_SEC })
// getPending: parse JSON; if exp < now return null
// cookie flags: HttpOnly; Secure; SameSite=Lax; Path=/authorize; Max-Age=300
```

Fingerprint helpers:

```ts
export function fingerprintFromOAuthReq(oauthReq: {
  clientId?: string;
  redirectUri?: string;
  codeChallenge?: string;
  [k: string]: unknown;
}): PendingAuth["fingerprint"] {
  return {
    clientId: oauthReq.clientId,
    redirectUri: oauthReq.redirectUri,
    codeChallenge: oauthReq.codeChallenge,
  };
}

export function fingerprintsMatch(
  a: PendingAuth["fingerprint"],
  b: PendingAuth["fingerprint"]
): boolean {
  return (
    a.clientId === b.clientId &&
    a.redirectUri === b.redirectUri &&
    a.codeChallenge === b.codeChallenge
  );
}
```
